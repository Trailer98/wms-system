# WMS RAG 知识库检索测试问题（v1-fixed）

覆盖范围：既有 10 条核心规则（其中 9 条已在 `wms-rag-knowledge-rules-v1-fixed.json` 的 `fixRules` 中给出修正后的完整正文）+ 本次新增 35 条规则（34 条业务规则 + 1 条新增的"库存字段映射规则"），共 45 条知识、每条至少对应 1 个测试问题；另附本次任务要求的 10 个必测问题（已分别归入对应模块表格并加粗标注"【必测】"）。

与 v1 版本相比的变化：

1. 财务相关问题的"期望回答要点"已改为明确要求"如实说明当前系统未实现财务事件功能"，不再允许回答里出现"通常会/不会生成财务事件"这类暗示功能已存在的表述。
2. 新增一条针对"库存字段映射规则"（`INVENTORY_FIELD_MAPPING_RULE`）的测试问题。
3. "出库单还没锁库，可以直接取消吗？"一题的预期命中知识调整为"出库取消解锁规则（fixRules 修正版，已包含未锁库取消场景）"，不再引用已被撤销的 updateRules 条目。

使用方式：在"RAG 问答测试"页面（`/ai/rag/ask` 或流式接口）逐条提问，对照"预期命中知识"检查 `references` 是否命中正确文档，对照"期望回答要点"检查 `answer` 内容是否准确、未编造实时数据、未使用 `on_hand_qty`/`locked_qty`/`frozen_qty`/`available_qty`/`total_qty` 这类不存在的字段名（技术人员模式下应使用 `quantity`/`reservedQuantity`/`frozenQuantity`/`getAvailableQuantity()`/`beforeQuantity`/`afterQuantity`）。

## 入库管理

| 问题 | 预期命中知识 | 期望回答要点 |
|---|---|---|
| **【必测】为什么入库后库存增加了？** | 入库收货与上架规则 | 收货是把货物正式登记为库存的动作；收货成功后现存库存（quantity）增加，并生成入库类型库存流水 |
| **【必测】入库为什么必须选择库位？** | 入库收货与上架规则 | 不选库位系统无法确定货物实际存放位置；创建入库单时必须为每个明细指定库区库位 |
| 入库单创建后能不能修改？ | 入库单状态流转规则 | 处于"已创建"状态可以编辑；已收货后不能再修改明细 |
| 入库单可以取消吗？ | 入库单状态流转规则 | 入库单没有"已取消"状态；未收货前可直接删除，已收货后不能删除 |

## 出库管理

| 问题 | 预期命中知识 | 期望回答要点 |
|---|---|---|
| 出库单从创建到发货要经过哪些状态？ | 出库单状态流转规则 | 已创建 → 已锁库 → 已发货，单向流转，已发货和已取消是终态 |
| **【必测】出库单为什么锁库失败？** | 出库锁库失败排查规则 / 出库库存不足处理规则 | 需检查该 SKU 全仓库可用库存是否足够、库位是否可用、是否位于异常区；失败会记录异常事件 |
| **【必测】库存有数量为什么还是不能出库？** | 锁定库存与冻结库存规则 / 可用库存计算规则（fixRules 修正版） | 现存库存不等于可用库存，需扣除锁定库存和冻结库存；出库判断的是可用库存（getAvailableQuantity()） |
| 已经发货的出库单能不能取消？ | 已发货出库单不能取消规则（fixRules 修正版） | 已发货出库单不能取消，应通过退货/红冲/反向业务处理 |
| 出库单还没锁库，可以直接取消吗？ | 出库取消解锁规则（fixRules 修正版，已包含未锁库取消场景） | 已创建（未锁库）状态取消无需释放锁定库存，直接改为已取消，不产生库存流水 |
| 出库确认发货后库存怎么变化？ | 出库确认发货规则（fixRules 修正版） | 扣减现存库存(quantity)和锁定库存(reservedQuantity)，生成发货流水，状态变为已发货 |

## 库存管理

| 问题 | 预期命中知识 | 期望回答要点 |
|---|---|---|
| 系统的库存数量字段都是什么意思？ | 库存数量字段规则说明（fixRules 修正版） | 现存(quantity)、锁定(reservedQuantity)、冻结(frozenQuantity)、可用(getAvailableQuantity())，不应出现 on_hand_qty 等旧命名 |
| **【新增】业务术语和代码字段是怎么对应的？** | 库存字段映射规则（INVENTORY_FIELD_MAPPING_RULE） | 现存库存→quantity，锁定库存→reservedQuantity，冻结库存→frozenQuantity，可用库存→getAvailableQuantity()，变化前后数量→StockMovementResponse.beforeQuantity/afterQuantity；并说明 on_hand_qty 等不是当前代码字段名 |
| 可用库存是怎么算出来的？ | 可用库存计算规则（fixRules 修正版） | getAvailableQuantity() = quantity - reservedQuantity - frozenQuantity |
| 同一个 SKU 在一个仓库会有几条库存记录？ | 库存余额唯一维度规则 | 按仓库+SKU+库区+库位四个维度唯一，不同库位分别产生独立记录 |
| 库存会不会出现负数？ | 库存不能为负规则 | 扣减操作都有原子校验，超量操作会被拒绝，不会出现负库存 |
| 锁定库存和冻结库存有什么区别？ | 锁定库存与冻结库存规则 | 锁定来自出库单占用；冻结来自异常区/盘点等场景；两者都不计入可用库存 |
| 库存的历史变化怎么查？ | 库存流水追踪规则 | 每次库存变化都会生成库存流水，记录变化前后数量（beforeQuantity/afterQuantity）和业务单号，可据此追溯 |
| 库存的正常/异常状态是怎么确定的？ | 库存状态与库区关系规则 | 库存状态随所在库区类型自动确定，进入异常区自动标记异常并冻结 |

## 库区 / 库位管理

|---|---|---|
| 仓库里有哪些类型的库区？ | 库区类型说明 | 收货区、存储区、拣货区、质检区、复核区、发货区、异常区、冻结区共 8 种 |

## 库存调整

| 问题 | 预期命中知识 | 期望回答要点 |
|---|---|---|
| 库存调增是怎么操作的？ | 库存调增规则 | 支持基于已有库存记录调增，或账外新增（需显式确认+指定库区库位） |
| 库存调减有什么要求？ | 库存调减规则（fixRules 修正版） | 不能超过可调减数量，破损/质量异常不能直接调减，需先转异常区；**不应提及"生成财务影响事件"** |
| 库存调整可以凭空调减一个不存在的库存吗？ | 调整必须基于库存记录规则 | 调减/转移必须选择已存在的具体库存记录，只有调增支持账外新增 |
| 库存调减最多能减到多少？ | 库存调减不能低于锁定和冻结库存规则 | 调减后现存库存不能低于锁定+冻结之和，已被占用的库存不能被普通调减扣除 |

## 异常管理

| 问题 | 预期命中知识 | 期望回答要点 |
|---|---|---|
| 质量异常的库存能直接调减吗？ | 质量异常库存处理规则 | 不能，与破损库存一样必须先转异常区，不能直接数量调减 |
| **【必测】破损库存为什么不能直接扣掉？** | 破损库存处理规则 | 实物还在仓库不应直接调减，需先转异常区再决定后续处理，否则账实不符 |
| 正常区的库存怎么转到异常区？ | 正常区转异常区规则（fixRules 修正版） | 是状态和库位归属变化而非调减，现存库存总量不变，源区可用库存减少；**不应提及"财务损益事件"** |
| **【必测】异常区库存怎么恢复正常？** | 异常区恢复正常区规则（fixRules 修正版） | 通过"恢复正常区"操作转回正常库区并解冻，需校验目标库位可用；**不应提及"财务损益事件"** |
| 异常区显示有库存，为什么出库还是分不到？ | 异常区库存不可直接出库规则 | 异常区库存自动冻结，可用库存为零，天然不会被出库分配选中 |
| 异常区的库存最终怎么从账上消掉？ | 异常区库存最终注销（报废）规则 | 没有独立报废按钮，需再发起一张库存调整单对该记录做数量调减 |

## 盘点管理

| 问题 | 预期命中知识 | 期望回答要点 |
|---|---|---|
| 盘点任务的范围可以怎么定？ | 盘点范围规则 | 可按仓库、或进一步按库区/库位缩小范围 |
| 盘点开始时系统记录的账面数量是什么时候的？ | 盘点快照规则 | 是开始盘点那一刻的库存快照，不会随后续变化被覆盖 |
| **【必测】盘点中库位为什么不能出入库？** | 盘点中库位禁止出入库规则 | 开始盘点会把涉及库位状态改为盘点中，期间入库出库调整都会被拒绝，完成/取消后恢复 |
| 盘点的实际数量怎么录入？ | 实盘数录入规则 | 任务必须处于盘点中状态才能录入，录入后明细状态变为已录入 |
| 盘点发现数量对不上怎么处理？ | 盘盈盘亏规则 | 实盘多于账面是盘盈、增加库存；少于账面是盘亏、扣减库存，都生成对应流水 |
| 盘点是什么时候真正改库存的？ | 盘点完成后才修正库存规则 | 只有全部明细录入实盘数量后点击"完成"才会修正库存，过程中不会 |

## 系统规则

| 问题 | 预期命中知识 | 期望回答要点 |
|---|---|---|
| 入库单号、出库单号是怎么生成的？ | 业务单号自动生成规则 | 前缀+8位日期+流水号自动生成，客户端指定的单号会被忽略 |
| 系统的操作日志会记录哪些信息？ | 操作日志记录规则 | 记录操作人、类型、模块、业务单号/对象、结果；日志失败不影响业务本身 |
| 系统是怎么控制不同岗位能看到的菜单和按钮的？ | RBAC权限控制规则 | 用户-角色-权限三层模型，权限点控制菜单/按钮可见性，接口同样校验 |
| 开发者角色和管理员角色是一回事吗？ | 开发者角色权限规则 | 不是同一个角色，管理员不会自动获得开发者的技术模式，需要显式分配 |
| 修改数据字典会影响出库这些业务逻辑吗？ | 数据字典使用边界规则 | 不会，字典只负责展示文案，业务逻辑以后端枚举为准 |

## AI / RAG 规则

| 问题 | 预期命中知识 | 期望回答要点 |
|---|---|---|
| **【必测】为什么 RAG 问答不能告诉我某个 SKU 当前库存？** | RAG只回答静态知识、不查询实时业务数据规则 | RAG 只检索静态知识库回答，不查询实时库存/订单/流水，会明确说明需通过业务查询获取 |
| 智能问答能不能自己去查一下库存再回答我？ | Tool Calling边界规则（当前未实现） | 当前系统没有接入 Tool Calling，不具备主动查询实时数据的能力 |
| **【必测】为什么开发者看到字段名，普通用户看不到字段名？** | 开发者与业务用户回答模式规则 | 回答模式由后端根据当前用户是否拥有 DEVELOPER 角色自动判断，不能手动切换或伪造 |
| 我改了知识库内容，为什么问答还是回答旧内容？ | 知识更新与接口变更后的知识维护规则 | 编辑后需要重新向量化（自动或手动触发），否则检索到的仍是旧向量 |
| **【必测】哪些库存操作会影响财务？** | 无对应已启用知识（预期检索不到财务相关规则） | **必须**明确回答"当前系统未实现财务事件功能，没有 finance_event 表或相关服务"，不应回答"调增/调减/盘盈/盘亏会生成财务事件"等暗示已实现的内容；若提及此话题应引导查看 `docs/generated/wms-rag-knowledge-design-candidates.md` 中的设计草案说明，而非当作现有功能作答 |
